Three Engines. One Query.

Stop gluing Pinecone + Neo4j + Postgres together. IADBMS unifies all three paradigms in a single UnifiedNode struct.

🧠

Vector Search (HNSW)

Native HNSW index with cosine similarity. Auto-embed text fields via Ollama — no Python, no glue code.

🔗

Graph Traversals (BFS)

Weighted edges with labeled relationships. Traverse 1..N depth in sub-millisecond. Built for agent memory networks.

📊

Relational Filters

BTreeMap-backed fields with O(1) bitset pre-filtering. Combine with vectors and graphs in a single IQL query.

🤖

Agent RBAC

Sub-graph isolation per agent role. Agent X can't see Agent Y's memory. Admin override. Zero-trust by default.

💬

Chat Primitives

INSERT MESSAGE USER "..." TO THREAD#5 — native conversational memory without JSON schemas or ORMs.

⚡

Zero-Copy I/O

Bincode over RocksDB pinned slices. No intermediate deserialization. Mechanical sympathy for NVMe and L1 cache.

See It In Action

From docker run to hybrid query in 30 seconds.

iadbms-cli
-- Insert a node (vector auto-generated via Ollama)
INSERT NODE#1 TYPE Persona {
  nombre: "Eros",
  bio: "Rust systems engineer"
}
✓ Node 1 inserted. Vector auto-generated (384d, 1.2ms).

-- Hybrid query: vector + graph + filter
FROM Persona
SIGUE 1..3 "amigo" Amigo
WHERE bio ~ "systems programming", min=0.85
FETCH nombre
✓ 3 results in 4.1ms

-- Native chat memory
INSERT MESSAGE USER "What is HNSW?" TO THREAD#1
✓ Message linked to Thread 1. Embedding stored.
