### Task 5/6: wal_shipping.rs compile errors (Win/Mac)

**Files:**
- Modify: `src/wal_shipping.rs`

**Fix:** Changed `-> Result<Self, Box<dyn Error>>` to `-> std::result::Result<Self, Box<dyn Error>>`. The `Result<T>` alias only takes 1 type param. This also fixes the `?` conversion of `reqwest::Error` because `Box<dyn Error>` implements `From<E>`.

**Verification:** `cargo test --features wal-shipping -- wal_ship` — 3 passed.

### Task 7: Miri UB Detection (miri_unsafe.rs)

**Files:**

**Fix:** Two fixes: (1) `NonNull::from(&data).as_ptr()` returns `*mut [f32; 5]`, cast to `*const f32`. (2) `SendPtr` test: heap-allocate data via `Box`, extract raw pointer before closure capture.

**Verification:** `cargo +nightly miri test --test miri_unsafe`.

### Task 10: MCP test panic (u128 JSON overflow)

**Files:**
- Modify: `src/sdk/serialization/graph_types.rs`
- Modify: `src/sdk/serialization/vector_types.rs`

**Fix:** Added `#[serde(with = "u128_serde")]` to `VantaNodeRecord.id`, `VantaEdgeRecord.target`, `VantaSearchHit.node_id`. XXH3-128 produces values > u64::MAX that serde_json cannot serialize as numbers.

**Verification:** `cargo test -p vantadb-mcp --test mcp_tests -- test_mcp_tool_search` — passed.
