### ⚠️ Exclusiones, Diferimientos y Justificaciones Técnicas

| Fase / Módulo | Motivo de Diferimiento / Exclusión | Justificación Técnica y de Producto |
| :--- | :--- | :--- |
| **OPS-01** (Docker / Server Hardening) | **Diferido Indefinidamente** | Contradice el principio fundamental "embedded-first / local-first". Añadir Docker/Server incrementa superficie de ataque, mantenimiento de imágenes y complejidad de red sin demanda validada. El server debe ser un wrapper opcional, no el centro del diseño. Solo activar si usuarios enterprise lo solicitan explícitamente tras consolidar adopción embebida. |
| **SEC-01** (Firma de binarios GitHub, SBOM) | **Diferido / Paralelo Opcional** | Útil para enterprise, pero no bloqueante para adopción inicial. PyPI ya usa Sigstore para wheels de Python. Puede añadirse como paso ligero en STAB-01 sin crear fase completa. No justifica inversión inmediata frente a necesidades de durabilidad y concurrencia. |
| **API-01** (Async SDK, Batch ops) | **Diferido** | Prematuro. El SDK síncrono es suficiente para agentes locales y casos de uso MVP. Async añade complejidad de GIL/PyO3 y concurrencia sin ROI inmediato. Solo justificado tras validar demanda real de throughput masivo en FEAT-01. |
| **Módulos Legacy / Experimentales** (LISP, IQL, governance, conflict resolver) | **Exclusión / Cuarentena** | Deuda técnica muerta. Módulos como `src/eval/`, `src/parser/lisp.rs`, `src/governance/` y `src/api/mcp.rs` (antes de refactorización) violan principio "embedded-first" y añaden superficie de ataque sin valor de producto validado. Deben moverse a `docs/legacy/` o crate separado con feature flag `experimental`. CI no debe compilarlos por defecto. |
| **Integración LLM/Ollama directa en core** | **Exclusión** | Contradice identidad "embedded-first". LLM/Ollama debe ser integración externa opcional, no dependencia del core. Solo justificado como ejemplo en `examples/` tras consolidar FEAT-01. |
| **Arquitectura Distribuida / Raft / Multi-Tenancy** | **Diferido Largo Plazo** | Prematuro y contradictorio con identidad "embedded-first". Solo justificado tras validar demanda real de enterprise y consolidar durabilidad/concurrencia en single-node. Complejidad extrema sin ROI inmediato. |
