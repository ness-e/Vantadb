# DRV-099: Haystack protocolo Document real

## Type
Rust/PyO3 integration — Haystack adapter

## Problem
`VantaDBDocumentStore` in `vantadb-haystack/src/python.rs` accepts/returns raw Python dicts instead of `haystack.dataclasses.Document` objects. Haystack pipelines fail because they pass `Document` instances and expect `Document` instances back.

## Solution
1. `write_documents`: accept both `dict` and `Document` objects — extract fields via duck typing (attrs for Document, keys for dict)
2. `filter_documents`: import `haystack.dataclasses.Document` at runtime and construct Document instances for results
3. Update `.pyi` type stubs
4. Add/update tests with real Document objects

## Contract
- `cargo check -p vantadb-haystack` passes
- Python tests pass with both dicts and Document objects

## Files
- `vantadb-haystack/src/python.rs` (main change)
- `vantadb-haystack/vantadb_haystack.pyi` (type stubs)
- `vantadb-haystack/tests/test_haystack.py` (test updates)
