## Contrato
"send() reintenta hasta 3 veces con backoff 1s→2s→4s en timeout antes de fallar"

## Investigation Notes
- Timeout actual: 5s, sin retry.
- OPFS `createWritable` puede bloquearse si otro tab tiene el mismo archivo abierto (File System Access API locking).
- Error de timeout se identifica por `message.contains("timeout")`.

## Steps

### Step 1: Agregar retry con exponential backoff en `send()`
- **Archivos:** `vantadb-wasm/src/worker.rs:186-247`
- **Acción:** extraer body actual a `try_send()`, crear `send()` wrapper con retry loop (max 3, backoff 1s→2s→4s)
- **Verify:** `cargo check -p vantadb-wasm --target wasm32-unknown-unknown`
- **Estado:** ✅ COMPLETED

**Resultado:** `send()` envuelve `try_send()` con retry loop (max 3, backoff 1s→2s→4s). Solo timeout errors reintentan. `cargo check -p vantadb-wasm` ✅.
