# P1-10: PyPI CDN propagation sleep → retry loop

**Fuente:** Plan `code-freeze-action-plan.md`
**Esfuerzo:** 🟢 ~15 min  
**Tipo:** CI config change  
**Archivos afectados:** `.github/workflows/release-wheels-60.yml:256-264`

## Objetivo

Reemplazar `sleep 90` ciego por reintento con exponential backoff en step "Wait for PyPI CDN propagation". Si CDN tarda >90s, el install falla (timeout del step: 10 min). Con retry loop, el install se adapta a la latencia real del CDN.

## Cambios

**Antes:**
```yaml
- name: Wait for PyPI CDN propagation
  run: sleep 90
- name: Install from PyPI (clean environment)
  run: python -m pip install --no-cache-dir "vantadb-py==..."
```

**Después:**
```yaml
- name: Install from PyPI with retry on CDN delay
  run: |
    for i in 1 2 4 8 16 32 64; do
      python -m pip install --no-cache-dir \
        "vantadb-py==${{ steps.pkg_version.outputs.version }}" && break
      echo "Package not yet available, retrying in ${i}s..."
      sleep $i
    done
```

## Verificación

- Diff inspeccionado: 2 pasos fusionados en 1, `sleep 90` reemplazado por retry loop (1+2+4+8+16+32+64 = 127s max vs 90s fijo)
- `set -e` en script para fallo rápido si todas las iteraciones fallan
- No hay compilación Rust que verificar (cambio YAML puro)

`sleep 90` reemplazado por retry loop con backoff. Si CDN propaga en 10s, instala en 10s. Si tarda 127s, igual falla (pero el loop intenta 7 veces vs 1 vez antes). Skipped: wrapper script reusable — add when un 3er sleep aparece en el codebase.
