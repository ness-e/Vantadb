# P0-1: CI `cancel-in-progress: true` mata jobs de test

**Fuente:** `docs/plans/code-freeze-action-plan.md`
**Estado:** ✅ Completado
**Fecha:** 2026-07-17

## Objetivo

`cancel-in-progress: true` a nivel workflow mata jobs de test/coverage/bench cuando hay push rápido. Mover a solo lint jobs (fmt, clippy).

## Archivos afectados

| Archivo | Cambio |
|---------|--------|
| `.github/workflows/ci-rust-10.yml` | Workflow-level `cancel-in-progress` → job-level solo en `fmt` y `clippy` |
| `.github/workflows/ci-web-11.yml` | `cancel-in-progress: true` → `false` (solo tiene build+test) |
| `.github/workflows/heavy-bench-nightly-51.yml` | `cancel-in-progress: true` → `false` |
| `.github/workflows/perf-bench-40.yml` | `cancel-in-progress: true` → `false` |
| `.github/workflows/sec-codeql-30.yml` | `cancel-in-progress: true` → `false` |

## Cambios realizados

- Agregado `concurrency: group: lint-${{ github.ref }}, cancel-in-progress: true` a job `clippy`
