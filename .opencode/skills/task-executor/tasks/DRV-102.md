## Blast Radius
- **Callers:** Python Langchain `add_texts()`, `similarity_search_by_vector()`, `delete()` in `integrations/langchain/vantadb_langchain/vectorstore.py`
- **Callees:** `VantaEmbedded::put()`, `VantaEmbedded::search()`, `VantaEmbedded::delete()` in core engine
- **API Changes:** None — `delete` gains `py: Python` but PyO3 injects it transparently; Python callers unchanged
- **Test Impact:** `vantadb-langchain/tests/test_langchain.py` covers all 3 methods

## Investigation Notes
- Pattern from PyO3 docs: `py.allow_threads(|| { blocking_code })` releases GIL during Rust-only work
- Other adapters (DSPy, Haystack) already use this pattern
- `delete` currently lacks `py: Python` — PyO3 injects it automatically when added to signature
